# Document Writing(文書作成)

## Structure(構成)
1. Conclusion first: state the main point in the first paragraph, details after(結論先出し).
2. Standard flow for proposals/reports: 背景 → 課題 → 提案/結果 → 効果/次のアクション.
3. One paragraph = one topic. One sentence = one idea. Split sentences longer than ~60 characters.
4. Prefer headings and lists over long prose. Numbered lists for sequences, bullets for parallel items. Three or more parallel data items → use a table.

## Tone & style(文体)
5. Pick 敬体(です・ます) or 常体(だ・である) from the existing document — or ask. Never mix within one document.
6. Cut redundancy:「〜することができます」→「〜できます」、「まず最初に」→「まず」、「〜という形になります」→「〜です」.
7. Conclusions get concrete subjects and assertions. Avoid vague hedges(「〜と思われる」「〜などが考えられる」)in conclusion sections.
8. Numbers, units, and dates: one consistent format throughout(半角数字; date format follows the document — YYYY-MM-DD or 和暦).

## Terminology(用語)
9. One concept = one term. Respect the document's existing terminology; never alternate synonyms(サーバ/サーバー、ユーザ/ユーザー).
10. Define abbreviations at first use.
11. Unknown facts are never filled in — use 【要確認】 per core rule 6.
